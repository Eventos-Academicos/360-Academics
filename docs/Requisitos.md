# 📄 Requisitos do Sistema - 360 Academics
## ✅ Requisitos Funcionais
- Cadastro de eventos  
- Gerenciamento de inscrições  
- Controle de presença  
- Emissão de certificados digitais  
- Cadastro de palestrantes e atividades  
- Histórico de participação  

## ⚙️ Requisitos Não Funcionais
- Segurança e autenticação  
- Interface intuitiva e responsiva  
- Escalabilidade para múltiplos eventos  
- Disponibilidade e confiabilidade  
- Aplicação baseada em ambiente **Web**

- ## 📌 Regras de Negócio
- Cada evento deve possuir um limite máximo de vagas definido pelo organizador.  
- Certificados só podem ser emitidos para participantes com presença confirmada.  
- Um participante não pode se inscrever em duas atividades que ocorram no mesmo horário.  
- Professores/palestrantes só podem emitir certificados vinculados às atividades que ministraram.
